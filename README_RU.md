# Музыкальный Магазин - Веб-приложение

Одностраничное веб-приложение, имитирующее музыкальный магазин с генерацией случайной информации о песнях, реалистичными обложками и воспроизводимой музыкой.

## Возможности

- **Поддержка языков**: Английский (США), Русский (Россия)
- **Генерация по seed**: Воспроизводимые данные на основе 64-битного seed
- **Динамическая система лайков**: Настраиваемое среднее количество лайков (0-10) с вероятностным распределением
- **Два режима просмотра**:
  - Табличный вид с пагинацией и раскрывающимися строками
  - Галерея с бесконечной прокруткой
- **Генерируемый контент**:
  - Названия песен, исполнители, альбомы и жанры на выбранном языке
  - Обложки альбомов с градиентными фонами и текстом
  - Воспроизводимые аудио-клипы с музыкальной структурой (аккорды, мелодия, гармония)
  - Текстовые фрагменты рецензий

## Технологический стек

- **Backend**: ASP.NET Core 8.0 (C#)
- **Frontend**: Vanilla JavaScript, HTML5, CSS3
- **Библиотеки**:
  - Bogus - генерация случайных данных
  - NAudio - генерация аудио
  - ImageSharp - генерация изображений

## Локальная разработка

### Требования

- .NET 8.0 SDK или выше
- Visual Studio Code или Visual Studio 2022

### Инструкции по установке

1. **Клонируйте репозиторий**:
   ```bash
   git clone <your-repo-url>
   cd MusicStoreApp
   ```

2. **Восстановите зависимости**:
   ```bash
   dotnet restore
   ```

3. **Запустите приложение**:
   ```bash
   dotnet run
   ```

4. **Откройте в браузере**:
   Перейдите на `http://localhost:5000` или `https://localhost:5001`

## Развертывание на Render

### Требования

- Аккаунт GitHub
- Аккаунт Render (доступен бесплатный тариф)

### Шаги развертывания

1. **Загрузите на GitHub**:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin <your-github-repo-url>
   git push -u origin main
   ```

2. **Создайте Web Service на Render**:
   - Перейдите на [Render Dashboard](https://dashboard.render.com/)
   - Нажмите "New +" → "Web Service"
   - Подключите ваш GitHub репозиторий
   - Настройте сервис:
     - **Name**: music-store-app (или на ваш выбор)
     - **Environment**: Docker
     - **Region**: Выберите ближайший к вам
     - **Branch**: main
     - **Instance Type**: Free (или выше для лучшей производительности)
   - Нажмите "Create Web Service"

3. **Дождитесь развертывания**:
   - Render автоматически соберет и развернет приложение
   - Первое развертывание занимает 5-10 минут
   - Вы получите URL типа: `https://music-store-app.onrender.com`

## Руководство по использованию

### Панель управления

1. **Region**: Выберите язык для генерируемого контента
2. **Seed**: Введите 64-битное число или нажмите "Random seed"
3. **Average Likes**: Установите значение от 0 до 10 (можно дробные числа)
4. **View**: Переключайтесь между режимами Table и Gallery

### Табличный вид

- Нажмите на любую строку, чтобы развернуть и увидеть:
  - Обложку альбома
  - Аудио-плеер
  - Текст рецензии
- Нажмите снова, чтобы свернуть
- Используйте кнопки Prev/Next для навигации

### Вид галереи

- Прокручивайте вниз для автоматической загрузки новых песен
- Каждая карточка показывает:
  - Обложку с градиентным фоном
  - Метаданные песни
  - Фрагмент рецензии
  - Аудио-плеер

## Технические детали

### Система seed

- Пользовательский seed + номер страницы → последовательная генерация данных
- Изменение seed перегенерирует весь контент
- Одинаковый seed всегда дает одинаковые результаты

### Генерация лайков

- Дробные значения используют вероятностное распределение
- Пример: среднее 3.7 = 70% получают 4 лайка, 30% получают 3 лайка
- Изменение лайков влияет только на количество лайков, а не на данные песен

### Генерация аудио

- Реальная синтезированная музыка (формат WAV)
- Воспроизводима по seed
- Музыкальная структура (не случайный шум)

### Генерация обложек

- Процедурные градиентные фоны
- Отрисованный текст с названием песни и исполнителем
- Уникальные цвета на основе seed

## Архитектура

- **Генерация на стороне сервера**: Все данные генерируются в памяти на сервере
- **Без базы данных**: Использует алгоритмическую генерацию
- **RESTful API**: JSON endpoints для получения данных
- **Без состояния**: Не требуется хранение сессий

## Производительность

- Песни генерируются по запросу
- Обложки кэшируются в браузере через base64
- Аудио стримится напрямую с сервера
- Бесконечная прокрутка загружает по 20 элементов за раз
